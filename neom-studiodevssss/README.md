# Neom Studio — منصة إدارة المطوّرين

موقع كامل (HTML + CSS + JavaScript) مربوط بقاعدة بيانات حقيقية عبر **Supabase**.
مفيش أي بيانات وهمية — كل حاجة بتتخزّن وتتقرأ من قاعدة البيانات.

---

## المحتويات (الفولدر)

```
neom-studio/
├── index.html            ← الصفحة الرئيسية (افتحها من خلال سيرفر — مش دبل كليك)
├── database.sql          ← سكربت قاعدة البيانات (تشغّله في Supabase)
├── README.md             ← الملف ده
├── css/
│   └── styles.css        ← التصميم
└── js/
    ├── config.js         ← ⬅️ الملف الوحيد اللي بتعدّله (المفاتيح)
    ├── supabase.js       ← الاتصال بقاعدة البيانات
    ├── ui.js             ← أدوات وواجهات مشتركة
    ├── store.js          ← الحالة + كل عمليات قاعدة البيانات
    ├── bus.js            ← إعادة رسم الواجهة
    ├── auth.js           ← الدخول / التسجيل / الانتظار / الحظر
    ├── profile.js        ← الملف الشخصي
    ├── tasks.js          ← مهام المطوّر
    ├── admin.js          ← لوحة الإدارة
    └── app.js            ← نقطة البداية والتوجيه
```

## خطوات التشغيل (بالترتيب)

### 1) اعمل مشروع على Supabase
- ادخل [supabase.com](https://supabase.com) واعمل حساب (مجاني، من غير فيزا).
- اضغط **New project** واختار اسم وباسورد لقاعدة البيانات، واستنى دقيقة لحد ما يجهز.

### 2) شغّل سكربت قاعدة البيانات
- من القائمة الجانبية اختار **SQL Editor** ← **New query**.
- افتح ملف `database.sql`، انسخ **كل** محتواه، والصقه في الصفحة.
- اضغط **Run**. ده هيعمل كل الجداول والتخزين تلقائيًا (مرة واحدة بس).

### 3) المفاتيح (اتحطّت لك بالفعل ✅)
قيمك متحطوطة خلاص في `js/config.js`، فمش محتاج تعمل حاجة هنا إلا لو غيّرت المشروع.

> **توضيح مهم بخصوص الروابط:**
> - **Project URL** الصح هو: `https://yuunstuyiberoqoupbjv.supabase.co` — اللي بينتهي بـ `.supabase.co` بس، **من غير** `/rest/v1/` في الآخر.
> - الرابط `https://neom-devs.omaryaro510.workers.dev` ده **مكان استضافة موقعك** (Cloudflare Workers)، مش رابط Supabase — متحطّهوش في config.

لو حبيت تغيّرهم يدويًا: Supabase ← Project Settings (الترس) ← **API**:

| القيمة | مكانها | تحطها في |
|--------|--------|----------|
| **Project URL** | API ← *Project URL* | `SUPABASE_URL` |
| **anon public key** | API ← *Project API keys* ← **anon / public** | `SUPABASE_ANON_KEY` |

> ⚠️ استخدم مفتاح **anon public** بس. متحطّش **service_role** في الموقع نهائيًا.

### 4) شغّل الموقع
الموقع بيستخدم وحدات ES modules، فلازم يفتح من خلال **سيرفر** (مش دبل كليك على `index.html`):

- **أسهل طريقة:** في VS Code نزّل إضافة **Live Server** ← كليك يمين على `index.html` ← *Open with Live Server*.
- **أو** من التيرمنال جوه الفولدر:
  ```bash
  python -m http.server 5500
  ```
  وبعدها افتح `http://localhost:5500`.
- **أو** ارفعه أونلاين مجانًا على **Netlify** أو **Vercel** أو **GitHub Pages** (اسحب الفولدر وخلاص).

## قواعد مهمة في النظام

- **حساب اسمه `Yaro`** بيبقى **مالك** (Owner) تلقائيًا ومُفعّل من غير ما حد يوثّقه — عشان تقدر تدير المنصة من أول حساب. (الشرط مضبوط داخل قاعدة البيانات نفسها في `database.sql`.)
- **ممنوع التكرار:** مفيش حسابين بنفس الاسم أو رقم الواتساب أو اسم روبلكس أو ديسكورد — مضبوط في الموقع وفي قاعدة البيانات.
- أي حساب جديد (غير Yaro) بيظهرله تنبيه إن حسابه **غير مُفعّل** لحد ما الإدارة تفعّله.
- المطوّر بيشوف **مهامه هو بس**. الإدارة بتشوف كل المستخدمين وكل المهام، وفيه زر **"عرض كل مهامه"** لكل مطوّر.
- **تسجيل الدخول محفوظ:** لو اللاعب قفل الموقع ودخل تاني بيفضل مسجّل دخول تلقائيًا. ولو حسابه لسه **تحت التوثيق**، بيفضل شايف رسالة "غير مُفعّل" مهما فتح وقفل، لحد ما الإدارة تفعّله. (زر **تسجيل الخروج** بيمسح الحفظ.)
- **حماية كلمة سر المالك:** أي حساب رتبته **مالك** كلمة سره **مش بتظهر** في لوحة الإدارة نهائيًا، حتى لو ضغطت "إظهار كلمات السر".
- **تحديث بيانات الإدارة:** لوحة الإدارة بتجيب أحدث البيانات كل ما تفتحها، وفيه زر **"تحديث"** فوق، وكمان بتتحدّث لوحدها لما ترجع لتبويب الموقع — فأي حساب جديد بيظهر فورًا.

## 🔒 ملاحظة أمان مهمة

عشان النظام يسمح للإدارة تشوف كلمات السر (زي ما طلبت)، الحسابات متخزّنة في جدول عادي وكلمة السر مكتوبة كما هي، والموقع بيوصل لقاعدة البيانات بمفتاح `anon` الظاهر في الكود.

معنى كده إن أي حد معاه رابط الموقع + المفتاح ممكن نظريًا يوصل للبيانات. ده **مقبول لأداة إدارة داخلية صغيرة**، لكن لو الموقع هيبقى عام لناس كتير أنصحك بترقية الأمان:

- تفعيل **Row Level Security (RLS)** بسياسات مضبوطة.
- استخدام **Supabase Auth** لتسجيل الدخول (وساعتها كلمات السر بتتشفّر ومش هتقدر الإدارة تشوفها).

لو حبيت أعملك النسخة الأكثر أمانًا دي (مع التخلي عن ميزة "عرض كلمة السر")، قوللي.

## مشاكل شائعة

- **"الجداول مش موجودة":** يبقى نسيت تشغّل `database.sql` (خطوة 2).
- **"تعذّر الاتصال" / "المفتاح غير صحيح":** راجع القيمتين في `js/config.js` (خطوة 3).
- **الصفحة بيضا أو الوحدات مش بتحمّل:** إنت فاتح الملف بدبل كليك — لازم سيرفر (خطوة 4).
- **الصور/الملفات مش بترفع:** اتأكد إن سكربت `database.sql` اتشغّل بالكامل (بيعمل bucket اسمه `uploads`).
